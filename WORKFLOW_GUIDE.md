# 🔄 개발 워크플로우 가이드

프론트엔드를 수정한 후 GitHub에 업로드하는 가장 편리한 방법들입니다.

---

## ⭐ 방법 1: GitHub Desktop (가장 추천!)

### 초기 설정 (1회만)

1. **GitHub Desktop 설치**
   - https://desktop.github.com/
   - 설치 후 GitHub 로그인

2. **저장소 추가**
   - File → Add local repository
   - `C:\Users\USER\Desktop\제주도 푸드맵` 선택
   - 원격 저장소 연결: `https://github.com/rlgnsdlsla1-ui/jeju-food-map.git`

### 이후 사용 (매번)

코드를 수정한 후:

1. **GitHub Desktop 열기**
2. **변경 사항 확인** (자동으로 표시됨)
3. **커밋 메시지 입력** (예: "메인 페이지 디자인 수정")
4. **Commit to main** 버튼 클릭
5. **Push origin** 버튼 클릭

**⏱️ 소요 시간: 10초!**

---

## ⭐ 방법 2: 배치 파일 (더블클릭 한 번!)

### 초기 설정 (1회만)

`quick-push.bat` 파일이 이미 준비되어 있습니다.

### 이후 사용 (매번)

코드를 수정한 후:

1. **`quick-push.bat` 더블클릭**
2. **끝!**

**⏱️ 소요 시간: 5초!**

---

## ⭐ 방법 3: VS Code / Cursor 내장 Git (에디터 안에서!)

### 초기 설정 (1회만)

이미 Git이 연결되어 있다면 바로 사용 가능합니다.

### 이후 사용 (매번)

코드를 수정한 후:

1. **Ctrl + Shift + G** (Git 패널 열기)
2. **변경 파일 옆 + 버튼** 클릭 (또는 Changes 옆 +)
3. **커밋 메시지 입력**
4. **✓ 버튼** 클릭 (커밋)
5. **... → Push** 클릭

**⏱️ 소요 시간: 15초!**

## 🤖 AI(저)가 도와드릴 수 있는 것

### ✅ 제가 할 수 있는 것

1. **코드 수정**
   - "메인 페이지에 새 섹션 추가해줘"
   - "버튼 색상을 파란색으로 바꿔줘"
   - 제가 코드 파일을 직접 수정해드립니다

2. **커밋 메시지 작성**
   - 어떤 변경사항인지 설명
   - 적절한 커밋 메시지 추천

3. **Git 명령어 생성**
   - 필요한 Git 명령어 제공
   - 배치 파일 업데이트

4. **배포 자동화 설정**
   - GitHub Actions 설정
   - Vercel 자동 배포 연동

### ❌ 제가 할 수 없는 것

1. **GitHub에 직접 푸시**
   - 보안상 사용자만 가능합니다
   - 하지만 클릭 한 번으로 가능하게 만들어드립니다!

## 🎯 권장 워크플로우

### 일반적인 개발 과정

```
1. AI에게 요청: "메인 페이지에 검색바 추가해줘"
   ↓
2. AI가 코드 수정 (자동)
   ↓
3. 로컬에서 테스트: npm run dev
   ↓
4. GitHub Desktop으로 푸시 (클릭 2번)
   ↓
5. Vercel 자동 배포 (자동)
   ↓
6. 완료! 🎉
```

**실제 작업 시간: 1분 이내!**

---

## 💡 더 자동화하고 싶다면?

### GitHub Actions로 자동 배포

코드를 푸시하면 자동으로:
- 빌드 테스트
- 린팅 체크
- 자동 배포

`github-actions.yml` 파일을 만들어드릴 수 있습니다!

### Vercel GitHub 연동

**이미 설정되어 있다면:**
- GitHub에 푸시만 하면
- Vercel이 자동으로 배포합니다
- 별도 작업 불필요!

## 🎁 보너스: 단축키 설정

### Windows에서 배치 파일 단축키 만들기

1. `quick-push.bat` 우클릭
2. **바로가기 만들기**
3. 바로가기 우클릭 → **속성**
4. **바로 가기 키:** `Ctrl + Alt + P`
5. **확인**

이제 어디서든 `Ctrl + Alt + P` → 즉시 푸시! ⚡
